MAIN MODULES & FEATURES
-------------------------

1. Authentication & Account Management
   - Email-based registration with password strength validation (12+ chars, uppercase, lowercase, digit, symbol)
   - Email confirmation flow — 6-digit OTP sent via Resend, 15-minute expiry
   - JWT session management — automatic session restore on app load via GET /api/profile
   - Login with account lockout — prevents credential stuffing via rate limiting and lockout tracking
   - User profiles — flexible JSON profile_data column with whitelist-based sanitization (ALLOWED_PROFILE_FIELDS)
   - Role-based access control — "job-seeker" and "recruiter" roles, protected routes enforce membership

2. Resume Management & Parsing
   - Two-step presigned upload flow — presign + finalize pattern for S3/R2 uploads
   - File validation — MIME type checking (PDF, DOCX), max 10MB
   - Deterministic rule-based resume parser (offline, no LLM):
     * Extracts experience, education, and skills from resume text
     * Section-scoped parsing (skills only from Skills section, etc.)
     * Date-range anchoring for experience entries
     * Handles PDF/DOCX text extraction with artifact normalization
   - Parser feedback loop — ResumeParseCorrection table tracks user edits for future improvement

3. Career Quizzes & Insights
   - Two structured quiz implementations: NextChapterYourWayQuiz, SolverahYourFutureYourWayQuiz
   - Quiz insights generation via OpenAI GPT-4o-mini with JSON structured output
   - Guest quiz preview — public quiz previews (no authentication required), rate-limited
   - Quiz data persistence — results stored in profile_data.quizInsights
   - Insight prompt engineering — custom system message instructs model to return JSON with exactly one insight per quiz

4. Job Recommendations (ML Pipeline - Gated)
   - Two-stage matching pipeline (company culture filter → skill match):
     * Stage 1: Culture-based company filtering (8 culture dimensions)
     * Stage 2: Job description vs. resume skill matching
   - Deterministic skill matcher (job_descr_LLM/matching/compare.py):
     * Dimensions: skills (0.5 weight), years_experience (0.2), certifications (0.1), education (0.1), clearance (0.1)
     * Jaccard-like overlap for skills, substring matching for education/clearance
   - Company culture dimensions (8 numeric/categorical):
     * Categorical: work_environment, pace
     * Numeric (0.0–1.0): empathy, creative_drive, adaptability, futuristic, harmony, data_orientation
   - Gated behind ML_ENABLED flag — disabled in demo mode to avoid external ML calls

5. Early-Access Intake Flow (Pre-Launch)
   - Lightweight early-access modal — first-name, last-name, email, optional phone/career journey
   - Full intake form submission — resume upload, personal details (state, phone, LinkedIn, portfolio)
   - Two-step submission — presign resume → finalize with personal data
   - Email notifications — sends to info@solverah.com on completion
   - Convert-to-account flow — /api/intake/create-account and /api/intake/sign-in endpoints
   - Link-to-account flow — /api/intake/link-account for existing users
   - IntakeSubmission table — persists submissions with status (uploaded, account_linked)

6. Data Seeding & Company Profiles
   - Company culture LLM module — generates/seeds company profiles with culture dimensions
   - Synthetic company data — generate_profiles.py builds CompanyProfile dicts, seeded via XLSX parsing
   - Culture narrative backfilling — backfill_culture_narratives.py populates narrative fields
   - Company archetype pipeline — company_archetype_pipeline/ for future pipeline expansion
   - Company model — stores name, slug, profile_json (full blob), narrative fields, 8 culture dimension columns

7. Audit Logging & Security Monitoring
   - AuditLog table — tracks action, resource, IP address, user agent, event metadata
   - Comprehensive event coverage — registration, login, profile updates, resume uploads, intake submissions
   - Metadata field — JSON column for contextual event data

KEY ARCHITECTURAL DECISIONS
-----------------------------

1. Two-Tier Authentication
   JWT in HttpOnly cookies (secure, automatic on each request) + CSRF token in response
   body for state-changing operations.

2. Decoupled ML
   ML modules (resume parser, skill matcher, company culture) are offline and deterministic
   by design — no external service dependencies. OpenAI is only used for quiz insights
   (optional, gated behind ML_ENABLED).

3. Presigned Upload Pattern
   Two-step S3 flow prevents server resource waste and allows the frontend to upload large
   files directly to Cloudflare R2 without proxying through the Flask server.

4. Profile Sanitization
   Whitelist-based cleaning of profile_data (ALLOWED_PROFILE_FIELDS) prevents over-posting
   attacks and keeps the JSON schema consistent.

5. Audit Trail
   Comprehensive logging of sensitive actions (registration, login, uploads, profile changes)
   for compliance, debugging, and security monitoring.

6. Flexible Company Data
   Company profiles stored as both full JSON blobs and indexed culture dimension columns
   to support both full retrieval and efficient filtering queries.

7. Feature Flags
   PRELAUNCH_MODE gates the landing page experience; ML_ENABLED + DEMO_MODE gate
   recommendation features to allow safe demo deployments.

8. Framework Composition
   React Context for auth state (simple, no Redux); custom hooks for form logic;
   Tailwind for styling (utility-first, no external component library).

9. Type Safety
   Full TypeScript on frontend; Pydantic models in ML pipelines for data validation.

10. Rate Limiting & Account Lockout
    Flask-Limiter + per-user lockout tracking protects against credential stuffing
    and brute force attacks while maintaining usability for legitimate users.
